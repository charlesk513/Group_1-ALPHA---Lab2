/******************************************************************************
** 1. Describe how your firstIndexOf() method in RangeBinarySearch.java
**    finds the first index of a key that equals the search key.
******************************************************************************/

The firstIndexOf() method uses a binary search to locate the first
occurrence of a term that matches the search key.

The algorithm compares the key with the middle element of the sorted
array using a comparator. If a match is found, the method records the index into the result variable
but continues searching on the left half of the array to check if there is an
earlier occurrence of the same key to be returned as the first occurance

This process continues until the search space is exhausted. The stored index
is then returned as the first position where the key appears. If no match is
found, the method returns -1 as our default value of result.


/******************************************************************************
** 2. What is the worst case time complexity in the number of compares
**    that each of the operations in the Autocomplete data type make,
**    as a function of the number of terms n and the number of matching terms m?
******************************************************************************/

sortDictionary():  [ O(n log n) ]

The dictionary is sorted using Arrays.sort() with a comparator. Sorting n
terms requires n log n comparisons in the worst case.

allMatches():  [ O(log n + m log m) ]

First, binary search is used twice to locate the first and last matching
prefix, which takes 2(O(log n)) comparisons and we ignore constatnts. Then the m matching terms are
copied and sorted by reverse weight order, which takes O(m log m).

numberOfMatches():  [ O(log n) ]

This method only performs two binary searches to find the first and last
occurrence of the prefix. No additional sorting or copying is performed.
